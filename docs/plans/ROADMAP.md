# Social App Roadmap (Web First)

更新时间：2026-03-05

## 1. 产品目标与范围

优先级（已确认）：
1. 聊天
2. 社群
3. 动态/朋友圈/Feed

MVP（Web）范围（已确认）：
- 必须有：私聊、群聊、官方社群、玩家自建社群、社群管理、拉黑玩家
- 暂不做：个人动态列表、Feed 推荐算法、视频内容
- 动态迭代顺序：纯文字 -> 图文 -> 视频（后置）

## 2. 当前基线（代码现状）

已具备：
- 账号系统（注册/登录/JWT/资料更新/头像上传）
- WebSocket 聊天（会话、消息、已读、私聊/群聊创建）
- 关系链（关注/粉丝/互关）
- 官方固定话题聊天室（内存态）

主要缺口：
- 后端测试基线不稳定（`go test ./...` 失败）
- “社群系统”尚未产品化（当前更像固定聊天室，不是可运营社群）
- 拉黑能力未落地
- 动态/Feed 尚未开始

## 3. 里程碑与时间线（建议）

## M0 稳定性基线（2026-03-09 ~ 2026-03-22）
- 修复后端测试（鉴权测试、proto 测试路径、sqlite/postgres 差异）
- 统一协议与测试数据约定
- 补齐关键监控日志（连接、消息、错误）
- 执行清单见 `docs/plans/M0_TASKS.md`

验收标准：
- `cd backend && go test ./...` 全绿
- `cd frontend && npm test` 全绿
- 核心聊天链路手工回归通过（登录->建会话->发消息->已读）

## M1 聊天 MVP 完整化（2026-03-23 ~ 2026-04-20）
- 私聊/群聊体验收敛：错误处理、重连恢复、消息一致性
- 拉黑系统 V1：数据模型、接口、WS 拦截（被拉黑后不可私聊/不可邀请）
- 聊天侧管理能力：会话成员管理（基础）

验收标准：
- 拉黑生效矩阵覆盖：发消息、建私聊、拉群邀请
- 双账号长连接稳定运行 24h 无致命错误

## M2 社群系统 V1（2026-04-21 ~ 2026-06-08）
- 社群模型落地：`communities / community_members / channels / bans`
- 官方社群持久化（替代当前内存固定房间）
- 玩家自建社群：创建、加入、退出、频道管理
- 权限体系：owner/admin/mod/member
- 管理动作：踢人、封禁、解封、频道管理

验收标准：
- 官方社群与玩家社群均可稳定聊天
- 社群管理动作可审计（操作日志）
- 权限越权测试通过

## M3 Web Beta 发布准备（2026-06-09 ~ 2026-06-22）
- 性能与可靠性优化（连接数、慢查询、错误率）
- 部署与回滚脚本完善（Jenkins + Docker）
- 最小运营面板/脚本（封禁、用户查询、消息排查）

验收标准：
- 可连续一周稳定运行
- 关键故障有可执行回滚路径

## 4. MVP 后路线（动态/Feed）

Phase A（优先）：纯文字动态（发帖、评论、点赞、时间流）
Phase B：图文动态（上传与审核）
Phase C：Feed 排序（先规则后算法）

## 5. 单人 + AI 执行方式

- 周节奏：周一计划、周二到周四开发、周五测试与修复
- 每周输出：可演示功能 + 测试结果 + 风险清单
- Definition of Done：
  - 功能可运行
  - 至少一条自动化测试
  - 文档更新（README/TEMP_STATUS/ROADMAP）
